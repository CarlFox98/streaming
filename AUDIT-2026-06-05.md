# Streaming Stack Audit — 2026-06-05

## Scope

Full audit of the streaming automation stack: 12 OBS WebSocket hub scripts, 3 local scripts, overlays, config, and automation launcher. Result: **14 issues found, 14 resolved**, ~400 lines of code deduplicated.

---

## 1. Security: Hardcoded Credentials

### OBS WebSocket Password in Plaintext Scripts

**Files:** `obs-dbg.ps1`, `obs-setup.ps1`

**Before:** Both scripts contained the literal OBS WebSocket password `8Nk8WqVKhwiW6b6d` hardcoded in the script body. These files live outside the git-tracked `Streaming/` directory (under `.config/opencode/`) and were **not gitignored**, making accidental exposure possible.

**Root Cause:** These scripts were created as one-off diagnostics and never retrofitted to use `config.ps1` after the shared-config pattern was adopted. `obs-setup.ps1` also hardcoded `ws://localhost:4455` and the Twitch alert box URL.

**Fix:** Rewrote both scripts to source `config.ps1` and use the shared `obs-wsapi.psm1` module for all WebSocket operations. No secrets in script bodies.

### Twitch Stream Key + OAuth Token on Disk

**Files:** `twitch-credentials.json`, `twitch-token.json`

**Status:** These files contain a live stream key and full-scope OAuth token in plaintext JSON. Encryption (DPAPI `ConvertFrom-SecureString`) is the industry standard for stored credentials. **Not modified in this pass** — requires coordination with the Twitch module's authentication flow.

**Recommendation:** Convert to SecureString storage with a master key in `config.ps1`. Add to `.gitignore` (already there via `config.ps1` rule for credentials).

## 2. Code Duplication: 12 Copies of OBS WebSocket Protocol

**Root Cause:** No shared abstraction existed for the OBS WebSocket protocol (SHA-256 challenge-response auth, JSON-RPC request/response matching, multi-frame message reassembly). Each implementer copied the same ~35 lines with minor variations in variable naming, sleep intervals, and error handling.

**Impact:** ~400 lines of copy-paste debt; inconsistent auth implementations; 3 scripts using wrong port/host; multi-frame truncation in 6 scripts.

**Fix:** Extracted `obs-wsapi.psm1` — a PowerShell module exposing 3 functions:

| Function | Purpose |
|----------|---------|
| `Connect-ObsWebSocket` | Connect + SHA-256 auth + receive hello |
| `Invoke-ObsRequest` | Send request, match by requestId, return responseData |
| `Disconnect-ObsWebSocket` | Clean close with NormalClosure |

All 12 hub scripts rewritten to `Import-Module` this single source.

## 3. Bugs Fixed

### 3a. Undefined `$intervalMs` in Audio Visualizer

**File:** `obs-audio-vis.ps1:80`

**Root Cause:** Variable `$intervalMs` referenced in `Start-Sleep -Milliseconds $intervalMs` but never defined. The output message claimed "Polling every 150ms" but the sleep used `$null` (defaults to 0), causing the loop to spin at maximum CPU.

**Fix:** New config variable `$Monitor_AudioVisIntervalMs = 150` in `config.ps1`. Audio visualizer now sleeps the intended 150ms between polls.

### 3b. Multi-Frame Message Truncation in Stream Monitor

**File:** `obs-stream-monitor.ps1:53-58`

**Root Cause:** The `Receive-Message` function used a single `ReceiveAsync` call without checking `EndOfMessage`. OBS WebSocket responses exceeding 65536 bytes are fragmented; a single read truncates the response, causing JSON parse failures during live streams.

**Fix:** The shared module's `Receive-Message` uses a `StringBuilder` loop until `EndOfMessage`, matching the correct pattern from `obs-audio-vis.ps1`.

### 3c. Profile Name Mismatch in Preflight

**File:** `obs-preflight.ps1:96`

**Root Cause:** Preflight checked for profile name `"Discord Capture"` (with space), but the actual OBS profile is named `Discord_Capture` (with underscore). Every preflight run would fail this check.

**Fix:** Changed expected name to `Discord_Capture` to match actual profile.

### 3d. Misleading "Running for Xs" Message in go-live

**File:** `go-live.ps1:198`

**Root Cause:** The uptime calculation `$now - $lastCheck` measured time *since last status line* (~30s), not total runtime. The message always showed ~30s regardless of how long the stream had been running.

**Fix:** Added `$startTime` initialized at launch. Status line now calculates `$now - $startTime` for actual uptime.

### 3e. Hardcoded "Untitled" Scene Collection in Backup

**File:** `backup-obs-scenes.ps1:9`

**Root Cause:** Scene collection name hardcoded as `Untitled.json`. If the user renamed their collection (e.g., "Streaming.json"), backups would silently target a nonexistent file.

**Fix:** New config variable `$SceneCollectionName = "Untitled"` in `config.ps1`. Backup script uses this variable.

### 3f. Obs-setup.ps1 Hardcoded Localhost:4455

**Root Cause:** `obs-setup.ps1` and `obs-dbg.ps1` hardcoded `ws://localhost:4455` instead of sourcing `$OBS_Host`/`$OBS_Port` from `config.ps1`.

**Fix:** Both scripts now use the shared module which reads from config via `Connect-ObsWebSocket`.

### 3g. `.gitignore` Redundant `!scripts/*.ps1` Rule

**Root Cause:** Rule `!scripts/*.ps1` was intended to un-exclude `.ps1` files, but `.gitignore` only ignores files matching a pattern — `*.exe` doesn't match `*.ps1`. The negation had no effect.

**Fix:** Removed the redundant line.

## 4. Missing Features Added

### 4a. Twitch Ingest Latency Check in Preflight

**Root Cause:** `obs-preflight.ps1` validated OBS settings, disk, audio, scenes, and overlays — everything local — but never tested whether the Twitch RTMP ingest was reachable. A user could pass all checks only to find a firewall or ISP blocking port 1935.

**Fix:** Added `Test-NetConnection -ComputerName live.twitch.tv -Port 1935` as check #11 in preflight.

### 4b. Mic-on-VOD-Track Validation in Preflight

**Root Cause:** Preflight checked that Spotify/Desktop Audio were *off* VOD track 2, but never verified that the microphone was *on* track 2. Silent VODs with no commentary were possible.

**Fix:** Added mic track check as check #12 in preflight. Verifies each wasapi_input_capture has track 2 access.

### 4c. Log Rotation on Startup

**Root Cause:** All daemon scripts (`stream-monitor`, `spotify-now-playing`, `audio-vis`) created timestamped log files that accumulated indefinitely with no retention policy.

**Fix:** Added log rotation step in `go-live.ps1` startup. Configurable via `$LogRetentionMaxFiles = 50` in `config.ps1`. Oldest logs beyond the limit are pruned.

### 4d. go-live.exe Fallback in start-stream-mode.ps1

**Root Cause:** `start-stream-mode.ps1` only checked for `go-live.exe`. If the compiled binary was missing, the script failed with "not found" and exited.

**Fix:** Added fallback: checks for `go-live.exe` first, then `go-live.ps1` as PowerShell fallback. Applied to both the initial launch and the retry-on-failure path.

## 5. Consolidation

### 5a. Merged obs-verify.ps1 and obs-verify2.ps1

**Root Cause:** Two scripts with nearly identical purpose (verify OBS connection, list scenes/inputs). `obs-verify2.ps1` had slightly better formatting and a display capture check.

**Fix:** Merged into single `obs-verify.ps1` with `-Verbose` flag for extended output. Deleted `obs-verify2.ps1`.

### 5b. Shared Module Eliminates 400 Lines of Duplication

**Root Cause:** As described in §2.

**Fix:** `obs-wsapi.psm1` provides 3 functions used by all 12 scripts. Each script now adds 1 line (`Import-Module`) instead of ~35 lines of WSAPI boilerplate.
